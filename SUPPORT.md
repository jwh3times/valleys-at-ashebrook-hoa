# Support

Thanks for using the Valleys at Ashebrook residents site! This page explains where to get help.
The site is a community project maintained by a single developer, so please be patient — but every
report and question is read.

## Before you ask

- Check the [README](README.md) for setup and local development, and [SETUP.md](SETUP.md) for the
  full Cloudflare deployment guide.
- Check [CONTRIBUTING.md](CONTRIBUTING.md) for the workflow and the checks a change must pass.
- Search [existing issues](https://github.com/jwh3times/valleys-at-ashebrook-hoa/issues?q=is%3Aissue)
  — your question may already be answered.

## Questions, bugs, and feature requests

Open an [issue](https://github.com/jwh3times/valleys-at-ashebrook-hoa/issues/new) with as much
context as you can. For bugs, include reproduction steps, expected vs. actual behavior, and the
page or endpoint where it happened.

**Residents:** for account or verification trouble (sign-up codes, roster mismatches), contact the
board through the site's contact form rather than GitHub — roster details are personal data and
don't belong in public issues.

## Security issues

**Do not** report security vulnerabilities through issues or pull requests. Follow the private
disclosure process in [SECURITY.md](SECURITY.md) (GitHub private advisory or
**<jerryholland00@gmail.com>**).

## Code of Conduct concerns

To report behavior that violates our [Code of Conduct](CODE_OF_CONDUCT.md), email
**<conduct@holland.vip>**.

## What to expect

The site is maintained in spare time, so there is no guaranteed support SLA. Bugs and questions get
a best-effort response, typically within about a week; security reports are acknowledged within a
few days per [SECURITY.md](SECURITY.md). If you don't hear back on a non-security issue after a
couple of weeks, a polite bump on the thread is welcome.
